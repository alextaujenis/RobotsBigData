Overview

Control many lights and blink without delay using commands such as on(), off(), blink(), and fade(). Lights must be connected to PWM enabled Arduino pins to work with this library.

Download

This library is part of a larger package that you can download to access all of the documented sensors, or you can use this library by itself with one dependency.

  • Download Light Library v2.0.2 
    .zip
     by itself with the Timer dependency, or
  • Download All Sensor Libraries with this library and dependency included, or
  • View Source on GitHub with MIT license
Example Wiring

  1. Connect the Arduino 
    ground pin
    to a resistor calculated for your LED
  2. Connect the resistor to the short (negative) LED leg
  3. Connect the Arduino 
    PWM pin 3
    to the long (positive) LED leg
  4. Load the Example Sketch onto the Arduino
  5. Watch the LED blink without delay

Public Methods

You can use this library without calling update() and most methods will function correctly. You must continuously call update() inside of loop() in order to use theReal-Time methods tagged below.

.update()

Keep calculating brightness values over time. This must be called continuously within loop() in order to blink() or fade() the light.

EXAMPLE SKETCH

.blink(on_time, off_time, [times])Real-Time

Pass in unsigned longs for the time values in milliseconds. Pass in an integer for the last parameter for the number of times the light should blink, which is [optional] and can be left out to make the light repeat the blink cycle forever.

  • the light will turn on for the on_time
  • then turn off for the off_time
  • and will repeat this cycle for the number of times you provided [optional]
EXAMPLE SKETCH
light
.fade(up_time, on_time, down_time, off_time, [times])Real-Time

Pass in unsigned longs for the time values in milliseconds. Pass in an integer for the last parameter for the number of times the light should fade, which is [optional] and can be left out to make the light repeat the fade cycle forever.

  • the light will perform a linear fade up according to the up_time
  • stay on for the on_time
  • linear fade down for the down_time
  • then stay off for the off_time
  • and will repeat this cycle for the number of times your provided [optional]
